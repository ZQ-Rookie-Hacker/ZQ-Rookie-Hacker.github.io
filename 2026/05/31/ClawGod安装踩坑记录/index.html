ClawGod 安装踩坑记录

字数: 计算中... 阅读时长: 计算中...

背景

ClawGod 是 Claude Code 的 runtime patch，对官方二进制进行运行时修补，解锁隐藏功能、移除各类限制。项目地址：github.com/0Chencc/clawgod。

核心能力：

  • Internal User Mode — 解锁 24+ 隐藏命令（/share, /teleport, /issue, /bughunter 等）
  • GrowthBook Overrides — 通过配置文件覆盖任意 feature flag
  • Agent Teams / Computer Use / Ultraplan / Ultrareview — 无需订阅
  • 移除安全限制 — CYBER_RISK_INSTRUCTION、URL 生成限制、谨慎操作确认
  • 绿色主题 — 一眼可知已 patched

坑 2：MSYS2 tar 与 Windows 路径不兼容

这是最麻烦的一个坑。设置代理后，安装脚本走 npm pack 路径下载完成，但 tar 解压时报错：

根因： Git Bash 的 MSYS2 tar（/usr/bin/tar）把 C:\... 路径解释为远程主机名。PowerShell 脚本调用 tar 时，PATH 中 Git Bash 的 tar 优先级高于 Windows 系统 tar。

解决： 调整 PATH 顺序，让 C:\Windows\System32\tar.exe 优先于 /usr/bin/tar。最终使用的完整命令：

关键点：

  1. Windows System32 必须在 MSYS2 /usr/bin 之前，确保 tar 走 Windows 原生版本
  2. Node.js / npm 目录（/c/myEnv/nodejs）必须在 PATH 中
  3. 代理变量通过 HTTPS_PROXY / HTTP_PROXY 传入，用于 npm 下载

坑 3：Git Bash 环境变量传递

从 Git Bash 调用 PowerShell 时，HTTPS_PROXY 和 HTTP_PROXY 需要 export 后由 PowerShell 继承。如果代理没传进去，npm 下载会超时或失败。

另外 install.sh 会检查 $OSTYPE 或 uname -s，在 MINGW64 下直接退出，所以不能在 Git Bash 中运行 bash 版本。

安装结果

25 个补丁全部应用成功，2 个跳过（Voice Mode 在当前版本无匹配、legacy filter 已不存在），0 失败。

使用

配置文件：

  • ~/.clawgod/provider.json — API key / baseURL 配置
  • ~/.clawgod/features.json — 功能开关

更新命令 claude update 已被 patch 重定向至 ClawGod 自更新流程，正常使用即可。

小结

整个过程的核心问题是 Windows 下 MSYS2 环境与原生 Windows 工具的 PATH 污染。调整 PATH 优先级，确保 System32 在 MSYS2 bin 之前即可解决。ClawGod 的安装脚本本身质量不错，自动完成二进制解析、cli.js 提取、native 模块提取、补丁应用，全程无需手动干预。
