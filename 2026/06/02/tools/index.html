0x01 dirsearch

执行说明

🔴 P0 — 每次必跑

1. 万能快速扫描

用途：第一轮快速过一遍，覆盖常见扩展名和敏感文件
注意：线程控制在 15 以下，加 delay 避免触发 CC 防护

2. 敏感文件/备份泄露专项

用途：专找备份文件、配置泄露、源码泄露，SRC 高危漏洞来源
注意：敏感路径探测容易触发 WAF，线程降到 10，delay 0.5s

3. Git/SVN 源码泄露

用途：.git/HEAD、.svn/entries 直接拿源码，P0 级漏洞
注意：Git 探测特征明显，务必加 delay 和随机 UA

🟡 P1 — 重要补充

4. 递归深度扫描

用途：发现子目录后自动深入扫描，适合大型站点
注意：递归扫描请求量大，线程降到 10，delay 0.5s 避免被封

5. 带认证扫描（登录后台）

用途：扫描需要认证才能访问的路径，找未授权访问、越权
注意：带认证扫描更敏感，线程控制在 10，加 delay

6. JS 文件/API 端点发现

用途：JS 中藏 API key、接口路径；API 端点直接找未授权接口
注意：API 端点探测容易触发风控，线程降到 10-15

7. 后台/管理路径扫描

用途：找管理后台入口、默认页面
注意：后台路径探测最容易触发封 IP，线程 10 + delay 0.5s

🟢 P2 — 深度挖掘

8. 自定义字典爆破

用途：默认字典不够用时，换大字典深度扫
注意：大字典扫描务必控制线程 10 + delay 0.8s，否则必被封

9. 多方法扫描

用途：PUT 上传点、OPTIONS 信息泄露
注意：PUT 方法探测特征明显，线程 10 + delay 0.5s

10. 状态码精细过滤

用途：403 绕过是 SRC 高频漏洞；301/302 可能暴露内部路径
注意：精细过滤扫描请求数多，线程控制在 15

🔵 P3 — 特殊场景

11. WAF/CC 绕过（重点）

用途：有 WAF/CC 防护时降低被封概率
核心策略：线程 5-10 + delay 0.5-2s + 代理池 + 随机 UA + 伪造 Header

12. 批量目标扫描

用途：SRC 批量资产扫描，一次性跑完所有子域名
注意：批量扫描更易触发 IP 封禁，务必线程 10 + delay 0.5s

13. 针对特定 CMS/框架

14. 输出格式与报告

0x02 ffuf

适用架构：Vue/React + Spring Boot、前后端分离、REST API
核心优势：支持 POST/JSON fuzz、HTTP/2、灵活过滤器，现代架构首选

执行说明

🔴 P0 — 每次必跑

1. Spring Boot Actuator 探测

用途：Actuator 是 Spring Boot 高危漏洞来源，env/heapdump/mappings 泄露敏感信息
注意：Actuator 端点通常有认证，关注 401/403 响应（存在但未授权）

2. Swagger/API 文档探测

用途：Swagger 泄露全部 API 接口、参数类型、认证方式，SRC 高危漏洞
注意：部分站点 Swagger 只在内网开放，外网探测关注 403

3. 基础 API 端点 fuzz

用途：发现未授权 API 端点、信息泄露接口
注意：现代 API 返回 JSON，用 -fs 0 过滤空响应

🟡 P1 — 重要补充

4. GET 参数 fuzz

用途：发现隐藏参数、未授权参数、SQL 注入点
注意：参数 fuzz 请求量大，线程 10 + delay 0.3s

5. POST/JSON body fuzz

用途：JSON body fuzz 是 Spring Boot API 的核心攻击面
注意：POST 请求更敏感，线程降到 5-10，delay 0.3-0.5s

6. HTTP 方法测试

用途：PUT/DELETE/PATCH 可能暴露越权操作、未授权访问
注意：方法测试特征明显，线程 10 + delay 0.3s

7. 认证绕过测试

用途：认证绕过是 SRC 高危漏洞，测试未授权访问、越权
注意：认证绕过测试请求量小，但特征明显，加随机 UA

🟢 P2 — 深度挖掘

8. IDOR/越权测试

用途：IDOR 是 SRC 最常见高危漏洞，遍历 ID 测试越权访问
注意：用普通用户 Token 测试，对比不同用户权限响应

9. 参数爆破 (字典)

用途：发现隐藏参数、未文档化接口
注意：参数字典要选对，Spring Boot 常用驼峰命名

10. 接口版本探测

用途：旧版本 API 可能存在已修复漏洞、未授权访问
注意：v1 通常比 v2 有更多安全问题

11. 敏感端点组合 fuzz

用途：发现管理后台、文件操作、调试端点等高危接口
注意：这些端点通常有认证，关注 401/403

🔵 P3 — 特殊场景

12. WAF/CC 绕过

用途：有 WAF/CC 防护时降低被封概率
核心策略：线程 3-5 + delay 0.5-1.5s + 代理池 + 随机 UA

13. 批量 API 扫描

用途：SRC 批量资产扫描，一次性跑完所有子域名的 API
注意：批量扫描更易触发 IP 封禁，务必线程 5 + delay 0.5s

14. 结果聚合分析

用途：聚合分析扫描结果，识别高价值目标
注意：401/403 端点可能存在认证绕过漏洞

0x03 feroxbuster

适用架构：Vue/React + Spring Boot、前后端分离、REST API
核心优势：Rust 高性能、自动校准过滤、智能递归、TUI 界面，大型目标首选

执行说明

🔴 P0 — 每次必跑

1. 基础 API 递归扫描

用途：递归发现 API 端点，自动学习响应模式减少误报
注意：--auto-tune 自动调整速率避免触发 WAF

2. 静态资源深度扫描

用途：发现 JS 源码、API 路径、敏感文件泄露
注意：静态资源扫描请求量大，控制线程 10 + delay 300ms

3. Spring Boot 端点扫描

用途：Spring Boot Actuator、Swagger、管理端点是高危漏洞来源
注意：Actuator 通常有认证，关注 401/403 响应

🟡 P1 — 重要补充

4. 带认证扫描

用途：扫描需要认证才能访问的 API，找未授权访问、越权
注意：带认证扫描更敏感，线程控制在 10，加 delay

5. 多方法扫描

用途：PUT/DELETE/PATCH 可能暴露越权操作、未授权访问
注意：方法测试特征明显，线程 10 + delay 300ms

6. 参数发现

用途：发现隐藏参数、未授权参数、SQL 注入点
注意：参数发现请求量大，线程 10 + delay 300ms

7. 扩展名组合扫描

用途：扩展名组合扫描发现隐藏接口、配置文件
注意：Spring Boot 常见 .do, .action 后缀

🟢 P2 — 深度挖掘

8. 深度递归扫描

用途：深度递归发现多层嵌套 API、隐藏目录
注意：深度递归请求量指数增长，delay 500ms + auto-tune

9. 自定义过滤器

用途：精确过滤减少误报，SPA 站点必须过滤 200 响应
注意：Vue SPA 未匹配路由返回 index.html (200)，需过滤响应大小

10. 大字典扫描

用途：默认字典不够用时，换大字典深度扫
注意：大字典扫描务必控制线程 10 + delay 500ms

11. 多维度组合扫描

用途：多维度组合扫描最大化覆盖
注意：组合扫描请求量大，务必 auto-tune + delay

🔵 P3 — 特殊场景

12. WAF/CC 绕过

用途：有 WAF/CC 防护时降低被封概率
核心策略：线程 3-5 + delay 500-2000ms + 代理池 + 随机 UA + auto-tune

13. 批量目标扫描

用途：SRC 批量资产扫描，一次性跑完所有子域名的 API
注意：批量扫描更易触发 IP 封禁，务必线程 5 + delay 500ms

14. 结果分析与去重

用途：聚合分析扫描结果，识别高价值目标
注意：401/403 端点可能存在认证绕过漏洞，大响应端点可能泄露信息
