'user'@'localhost' 仅本机连接 本地管理用户（安全）

2.作用域

作用域格式 示例 说明
*.* grant select on *.* 所有数据库的所有表
数据库名.* grant insert on sales.* 指定sales数据库的所有表
数据库名.表名 grant update on hr.employees 指定hr数据库和employees表
列名(特殊语法) grant select (id,name), update (salary) ON hr.employees 列级权限 （指定库与表）

3.WITH 选项（小写）

选项名称（小写） 作用 示例
with grant option 允许用户将自己的权限授予其他用户 grant select on db.* to 'user'@'%' with grant option;
with max_queries_per_hour n 限制每小时允许的查询次数 grant select on db.* to 'user'@'%' with max_queries_per_hour 1000;
with max_updates_per_hour n 限制每小时允许的更新次数 grant update on db.* to 'user'@'%' with max_updates_per_hour 500;
with max_connections_per_hour n 限制每小时允许的连接次数 grant all on db.* to 'user'@'%' with max_connections_per_hour 100;
with max_user_connections n 限制同时在线的用户连接数 grant all on db.* to 'user'@'%' with max_user_connections 5;
with require ssl 强制用户使用 SSL 加密连接 grant all on db.* to 'user'@'%' with require ssl;
with require x509 强制用户使用 X.509 证书认证 grant all on db.* to 'user'@'%' with require x509;

四、用户权限查询

1.权限概览

（1）权限层级

层级 作用范围 存储表 查询方法
全局级 所有数据库 mysql.user show grants for user@host
数据库级 单个数据库 mysql.db select * from mysql.db
表级 单个表 mysql.tables_priv select * from mysql.tables_priv
列级 表中特定列 mysql.columns_priv select * from mysql.columns_priv
存储过程级 存储过程/函数 mysql.procs_priv select * from mysql.procs_priv

（2）解释

1.所谓的全局查询，就是使用查询语句show grants 查询用户的权限信息， 其查询结果的信息条数不定，每一条查询结果就对应一次赋权（当然，用户可以被一次性赋予多个权限，此处讨论的是用户被·赋权的次数），全局查询结果包含各种级别的权限信息（或存在权级覆盖）
2.所谓数据库级查询，是查询用户对数据库的操作权限

2.全局级权限查询

（1）使用show grants

权限查询结果分析：

  1. GRANT 是授予权限的关键字
  2. ALL PRIVILEGES 表示所有权限（像对数据库、表的增删改查、创建、删除等各类操作权限 ）
  3. ON *.*中，第一个 * 代表所有数据库，第二个 * 代表对应数据库里的所有表，即该用户对服务器上所有数据库的所有表都有操作权限。
  4. TO 'root'@'localhost'：指定权限授予的对象是从本地主机（localhost）登录的 root 用户
  5. IDENTIFIED BY PASSWORD '*81F5E21E35407D884A6CD4A731AEBFB6AF209E1B'：这是存储的密码哈希值，MySQL 不会明文存储密码，而是通过哈希算法处理后存储，用于验证用户登录时输入的密码是否正确，这里展示的就是 root 用户密码经过哈希后的结果
  6. WITH GRANT OPTION：意味着该用户不仅自己拥有这些权限，还能够将自己所拥有的权限授予其他用户，比如可以执行 GRANT 语句给其他 MySQL 用户分配权限
  7. PROXY 权限允许用户代表其他用户进行操作
  8. "@'' 这种形式在 MySQL 里可理解为涵盖了各类符合条件的用户场景

7.权限覆盖

现有：

  • 数据库：test_db
  • 表：users、orders
  • 用户：app_user

app_user
赋权:

USAGE：允许用户连接到 MySQL 服务器，但无任何数据库操作权限。
SELECT, INSERT, UPDATE：用户可以查询、插入和更新所有数据库的表。

1.全局权限被数据库级权限覆盖

数据库范围 SELECT 权限 INSERT 权限 UPDATE 权限 解释说明
test_db ✅ 有 ❌ 无 ❌ 无 数据库级 SELECT 权限覆盖了全局的同名权限，并移除了全局的 INSERT/UPDATE。
其他所有数据库 ✅ 有 ✅ 有 ✅ 有 全局权限 (SELECT, INSERT, UPDATE ON *.*) 仍然有效。

2.全局权限被表级权限覆盖

权限范围 数据库/对象 SELECT INSERT UPDATE 解释说明
全局权限 *.* ✅ 有 ✅ 有 ✅ 有 基础权限：所有数据库的所有表
表级权限 test_db.users ✅ 有 ❌ 无 ❌ 无 表级权限覆盖了全局权限在特定表上的 INSERT/UPDATE
其他数据库表 非 test_db ✅ 有 ✅ 有 ✅ 有 全局权限生效
test_db 其他表 test_db.* (除 users) ❌ 无 ❌ 无 ❌ 无 没有显式授权，因此没有任何权限

3.全局权限被列级覆盖

app_user被授予全局查询权限

app_user被授予表级查询权限

操作场景 能否执行 依赖权限 原理说明
查询 test_db.users(id) ✅ 允许 列级+全局 列级明确允许，无冲突
查询 test_db.users(create_date) ❌ 拒绝 列级限制 列级未授权，覆盖全局权限
查询 test_db.orders(*) ✅ 允许 全局 SELECT ON *.* 无更细粒度限制，全局生效
查询 other_db.other_table ✅ 允许 全局 SELECT ON *.* 无更细粒度限制，全局生效
