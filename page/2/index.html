0x01.基于源站反射的 CORS 漏洞

该网站存在不安全的 CORS 配置，因为它信任所有源站。
要完成本实验，你需要编写一段 JavaScript 代码，利用 CORS 漏洞获取管理员的 API 密钥，并将该代码上传至你的漏洞利用服务器。成功提交管理员的 API 密钥后，实验即告解决。
你可以使用以下凭据登录自己的账户：wiener:peter

增加Origin字段

  • Access-Control-Allow-Origin：允许谁访问

    存在两种形式：

    1. 具体源 只允许这一个源访问，其他都拒绝 如：Access-Control-Allow-Origin: https://web.example.com
    2. 通配符 * 允许任意源访问，但不允许携带凭证 Access-Control-Allow-Origin: *

  • Access-Control-Allow-Credentials：是否允许携带凭证

    允许这个跨域请求携带用户的敏感凭证（如 Cookie、HTTP 认证信息、TLS 客户端证书）。
    缺失表示不允许携带凭证（默认行为）

再次修改，可以证明存在CORS漏洞

浏览器同源策略与CORS解析

post @ 2026-04-12

前言：为什么需要同源策略？

互联网诞生之初，浏览器允许任意网页随意访问其他网站的数据。这带来了巨大的安全隐患：恶意网站可以悄悄读取你在银行网站上的个人信息，甚至冒充你进行操作。为了解决这个问题，浏览器引入了 同源策略，作为一道基本的安全防线。

但同源策略过于严格，现代 Web 应用又需要合法的跨域通信（比如前端调用不同域名的 API）。于是，W3C 制定了 CORS，在保证安全的前提下，为跨域访问打开一扇可控的门。

本笔记将按照 “问题 → 解决方案 → 细节机制 → 安全风险” 的逻辑，为你彻底讲清同源策略与 CORS。

1. 同源策略：浏览器安全的基石
