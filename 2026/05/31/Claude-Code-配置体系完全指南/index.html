一、概述：你的 Claude Code 定制全貌

序号 能力 实现方式

三、MCP Server 配置体系

3.1 核心结论

Claude Code 的 User MCP 只从 .claude.json 读取，不是从 settings.json。

在 Claude Code 中输入 /mcp，看到的 “User MCPs (C:\Users<Username>.claude.json)” 就是来自这个文件。

3.2 两个文件的 MCP 字段区别

.claude.json mcpServers .claude/settings.json mcpServers
作用 User 级全局 MCP，Claude Code 直接读取 Provider 绑定型 MCP，CC Switch 写入
包含内容 所有 MCP Server（time/memory/context7/…） 仅 provider 专属的（如 exploit-server 带 token）
修改方 CC Switch 同步 / 手动编辑 CC Switch 从 provider 的 settings_config 生成
Claude Code 读取 ✅ 是（User MCPs 来源） ❌ 否（仅作 provider 配置）

3.3 数据流：CC Switch → 磁盘文件

⚠️ 同步方向是单向的：CC Switch DB → 磁盘文件。CC Switch 是数据源（source of truth），直接改磁盘文件可能被 CC Switch 下次同步覆盖。

3.4 CC Switch 数据库内部结构

关键区分：

  • providers.settings_config.mcpServers → 写入 settings.json（provider 专属 MCP，如带 API Token 的）
  • mcp_servers 表（全局） → 写入 .claude.json（所有 User MCP）
  • settings.common_config_claude → 写入 settings.json（hooks 等公共配置）

3.5 当前 7 个 MCP Server 一览

MCP Server 类型 用途
time stdio (uvx) 时间查询 / 时区转换
memory stdio (npx) 知识图谱记忆系统
context7 stdio (npx) 实时文档查询
sequential-thinking stdio (npx) 链式思维推理
fetch stdio (uvx) HTTP 请求抓取
exploit-server stdio (python) Exploit Server Pro（Payload/日志/OOB/Repeater）
burp stdio (java proxy) Burp Suite 联动（SSE 代理）

3.7 新增 MCP Server 操作流程

方法 A（推荐）：CC Switch UI 添加

  1. 打开 CC Switch → MCP Servers 管理
  2. 点击 “Add MCP Server”，填写配置
  3. 勾选 enabled_claude = true
  4. CC Switch 自动写入 DB + 同步到 .claude.json
  5. 重启 Claude Code 或 /mcp reload

方法 B：手动编辑 .claude.json

  1. 编辑 C:\Users\<Username>\.claude.json，在 mcpServers 中添加条目
  2. 重启 Claude Code
  3. ⚠️ 注意：下次 CC Switch 同步可能覆盖手动修改（需要在 CC Switch DB 中也插入对应记录）

方法 C：同时写入两处（最稳妥）

  1. 编辑 .claude.json 添加 MCP
  2. 在 CC Switch DB 的 mcp_servers 表中也插入一条
  3. 两边数据一致，互不覆盖

四、Skill 技能体系

4.1 技能的双轨识别机制

你的 102 个安全技能通过两条独立的路径被 Claude Code 识别：

4.2 两种识别方式对比

维度 轨道 1：原生 Skill 机制 轨道 2：自定义 Hook 路由
触发方式 /skill-name 命令 或 Skill tool 自动触发 自然语言输入 → 关键词自动匹配
执行方式 子 Agent 独立执行 文本注入当前上下文（内联）
多技能 一次一个 一次推荐最多 3 个，标注互补关系
中文支持 有限 ✅ 50+ 组中英双语别名
匹配策略 静态规则（文件名/frontmatter） 两相打分：Phase 1 精确别名 + Phase 2 模糊关键词
优先级调节 无 基于使用频次/时间的动态权重
CC Switch 管理 ✅ 通过同步 ✅ 直接管理源目录
维护成本 低（放目录即可） 需维护脚本 + 别名库 + 索引

4.3 Hook 架构详解

你的 Hook 体系在 UserPromptSubmit 事件上挂载了两个脚本：

Hook 脚本位置：

文件 作用
~/.claude/hooks/security-context-hook.py LLM 场景分类，注入 [security:xxx] 标签
~/.claude/hooks/skill-router-hook.py 关键词匹配 + 技能推荐，注入文件路径
~/.claude/hooks/build-skill-index.py 扫描 SKILL.md → 构建 .skill-index.json
~/.claude/hooks/keyword-aliases.json 中英双语关键词别名库（50+ 组）
~/.claude/hooks/status-line.py 终端状态栏回显 Hook 匹配结果

Hook 关键约束：

  • Hook 只能通过 additionalContext 注入文本，不能直接调用 Skill tool
  • 推荐是”建议”而非”强制执行”——Claude 看到推荐后手动 Read
  • 可以在 additionalContext 中用更强指令（如 “你必须立即使用 Read tool 读取以下文件”）模拟自动调用

4.4 CC Switch 与 Skill 的关系

操作 CC Switch 作用
安装技能 CC Switch UI 一键安装，写入 ~/.cc-switch/skills/
更新技能 CC Switch 检测上游更新并同步
同步到 Claude Code CC Switch 将技能复制到 ~/.claude/skills/，使 /skills 命令可见
Hook 路由源 Hook 从 ~/.cc-switch/skills/ 扫描技能，做关键词匹配推荐

当前状态：

两者基本一致（CC Switch 同步后）。Hook 的 skill-router-hook.py 和 build-skill-index.py 已修改为指向 ~/.cc-switch/skills/。

4.5 技能同步机制

CC Switch 中的 skillSyncMethod 设为 auto 时：

  1. CC Switch 维护 ~/.cc-switch/skills/ 作为源目录
  2. 同步时复制到 ~/.claude/skills/（Claude Code 原生目录）
  3. 每个技能是 目录名/SKILL.md 格式，带 YAML frontmatter

⚠️ 遇到过同步不完整的情况：auto 模式下只同步了 1 个。解决方案：在 CC Switch GUI 中手动触发一次全量同步。

6.3 排错流程

问题 检查点
MCP 不出来 ① .claude.json 中是否有该条目 ② /mcp reload ③ CC Switch 是否覆盖了手动修改
Skill 不被识别 ① ~/.claude/skills/ 是否存在 ② SKILL.md frontmatter 格式 ③ CC Switch 是否同步
Hook 不推荐 ① keyword-aliases.json 是否有对应关键词 ② .skill-index.json 是否最新 ③ 脚本路径是否正确
CC Switch 覆盖手动修改 手动修改 .claude.json 后，需同步到 CC Switch DB，否则下次同步丢失

七、常见误区

误区 真相
“改 settings.json 就能加 MCP” ❌ Claude Code 的 User MCP 读 .claude.json，不是 settings.json
“CC Switch DB 改了就能直接生效” ❌ 需要同步到 .claude.json（CC Switch 自动或手动触发）
“.claude.json 是 CC Switch 的输入” ❌ 它是 CC Switch 的输出目标，会被覆盖
“settings.json 的 mcpServers 是 User MCP” ❌ 那是 provider 专属 MCP，User MCP 在 .claude.json
“所有 MCP 都在一个文件” ❌ provider 绑定型在 settings.json，全局共享型在 .claude.json
“Skill 只能通过 / 调用” ❌ 你有双轨：原生 /skills + Hook 自动推荐
“Hook 可以自动调用 Skill” ❌ Hook 只能注入文本推荐，Claude 需要手动 Read
“CC Switch 同步是双向的” ❌ 单向：CC Switch DB → 磁盘文件
