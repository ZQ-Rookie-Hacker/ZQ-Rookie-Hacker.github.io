1. 系统基本操作

系统控制

  • reboot - 重启系统
  • shutdown -h now - 立即关闭系统

  • echo $JAVA_HOME - 查看JAVA_HOME环境变量
  • echo $PATH | grep jdk - 检查PATH中是否包含JDK路径

用户切换与管理

  • su 用户名 - 切换到普通用户
  • logout - 注销当前用户
  • passwd - 修改当前用户密码

历史命令

  • history - 查看历史命令
  • !! - 重复上一条命令
  • !n - 重复第n条命令

目录导航

  • pwd - 显示当前工作路径
  • cd .. - 返回上一级目录
  • cd ../.. - 返回上级两级目录
  • cd - - 返回上次所在的目录
  • cd 文件夹名 - 进入指定文件夹
  • cd ~ - 进入当前用户主目录

文件操作

  • touch 文件名 - 创建空文件
  • cat 文件名 - 读取文件内容
  • mkdir 文件名 - 创建文件夹
  • mkdir -p 文件夹 - 递归创建文件夹
  • mkdir 文件名 文件名 - 创建多个文件夹
  • cp 源文件 目标文件 - 复制文件
  • cp -r 源文件夹 目标文件夹 - 递归复制文件夹
  • mv 旧文件名 新文件名 - 移动/重命名文件
  • rm 文件 - 删除文件
  • rm -f 文件 - 强制删除文件

  • rm -rf 文件/文件夹 - 强制递归删除文件/文件夹及其内容

文件搜索

  • find / -name file1 - 从根目录开始搜索文件和目录
  • find / -user user1 - 搜索属于指定用户的文件和目录
  • find /home/user1 -name *.bin - 在指定目录中搜索特定扩展名的文件
  • find /usr/bin -type f -atime +100 - 搜索过去100天未被使用的执行文件
  • find /usr/bin -type f -mtime -10 - 搜索10天内被修改过的文件
  • find . -name '.php' -mmin -30 - 查找最近30分钟修改的当前目录下的.php文件
  • find . -name '*.inc' -mtime 0 -ls - 查找最近24小时修改的文件并列出详细信息
  • find / -name *.rpm -exec chmod 755 '{}' \; - 搜索并修改权限
  • find / -xdev -name *.rpm - 搜索忽略可移动设备
  • locate *.ps - 快速查找以.ps结尾的文件（需先运行updatedb）
  • whereis halt - 显示二进制文件、源码或man的位置
  • which halt - 显示二进制文件或可执行文件的完整路径
  • find / -name moonsec 2>/dev/null - 搜索文件/文件夹并屏蔽错误信息

6. 网络相关命令

网络相关文件

注意

  • 现代kali的网卡主管理工具是NetworkManager
  • 配置文件位置：/etc/NetworkManager/system-connections/（每个连接一个 .nmconnection 文件）
  • 传统方式是通过编辑/etc/network/interfaces文件来配置网络接口

  • /etc/network/

    • 用途：传统的 Debian 网络配置目录（ifupdown 系统）。
    • 关键文件：interfaces（定义静态 IP、DHCP 等）、interfaces.d/（子接口配置）。
    • 现状：在 Kali 桌面版中，逐渐被 NetworkManager 取代，但在服务器版或某些特定配置中仍在使用。

  • /etc/NetworkManager/

    • 用途：NetworkManager 服务的配置目录。Kali 桌面版默认使用此服务管理网络。
    • 关键文件：NetworkManager.conf、system-connections/（存储具体的 Wi-Fi 和有线连接配置文件，含密码）。

  • /etc/systemd/network/

    • 用途：如果使用 systemd-networkd 管理网络，配置文件存放于此。
    • 关键文件：.network, .netdev, .link 文件。

  • /etc/netplan/

    • 用途：主要用于 Ubuntu，但在某些 Kali 变体或特定云镜像中可能出现。用于生成 backend (NetworkManager 或 systemd-networkd) 配置。

  • /etc/dhcp/

    • 用途：DHCP 客户端配置文件。
    • 关键文件：dhclient.conf（控制 DHCP 请求行为，如 DNS 覆盖等）。

  • /etc/wpa_supplicant/

    • 用途：Wi-Fi 加密认证配置。
    • 关键文件：wpa_supplicant.conf（命令行连接 Wi-Fi 时使用）。

  • /etc/resolv.conf

    • 用途：配置 DNS 服务器地址。
    • 注意：在現代系统中，该文件通常由 NetworkManager 或 systemd-resolved 自动生成/管理，手动修改可能会被覆盖。

  • /etc/hosts

    • 用途：本地静态主机名解析（IP 到域名的映射），优先级通常高于 DNS。

  • /etc/hostname

    • 用途：存储本机的主机名。

  • /etc/nsswitch.conf

    • 用途：名称服务切换配置。决定系统查找主机名、用户、组等的顺序（例如：先查 files 即 hosts，再查 dns）。

  • /etc/ethers

    • 用途：将以太网 MAC 地址映射到 IP 地址（较少用，需配合 arp 守护进程）。

  • /etc/services

    • 用途：定义网络端口号与服务名称的对应关系（例如：80 对应 http，22 对应 ssh）。

  • /etc/protocols

    • 用途：定义 IP 协议号（如 TCP, UDP, ICMP 等）。

  • /etc/ssh/

    • 用途：SSH 服务配置（网络远程访问核心）。
    • 关键文件：sshd_config（服务端配置）、ssh_config（客户端配置）。

  • /etc/iptables/

    • 用途：保存 iptables 防火墙规则的文件（如果使用 iptables-persistent）。

  • /etc/nftables/

    • 用途：保存 nftables 防火墙规则（现代 Linux 逐渐转向 nftables）。 /etc/ufw/
    • 用途：如果使用 UFW (Uncomplicated Firewall)，配置在此目录。

  • /etc/networkd-dispatcher/

    • 用途：当网络状态改变时触发脚本的目录。

  • /etc/NetworkManager/dispatcher.d/

    • 用途：NetworkManager 在网络连接/断开时执行脚本的目录。
  • /etc/proxychains4.conf
    • 用途：配置代理服务器（如 SOCKS4/5, HTTP 等）。
    • 关键文件：proxychains4.conf（全局配置）、proxychains.conf（旧版配置）。

7. 服务管理

systemctl 是 systemd 系统（Kali Linux、Debian、Ubuntu、CentOS 7+ 等主流Linux均采用）的核心服务管理工具，替代了旧版的 service、chkconfig、/etc/init.d/ 等命令，是服务管理的首选方式。

7.0 systemd 核心单元类型说明

systemd 以「单元（Unit）」为核心管理系统资源，常见单元类型（后缀）如下：

  • service：常规后台服务（如cron.service、clamav-daemon.service），长期运行的守护进程，处理持续的系统功能；
  • socket：套接字服务（如 Kali 的ssh.socket），轻量监听端口/套接字，仅当有连接请求时才唤醒对应进程（大幅节省系统资源，Kali 中广泛用于低负载服务）；
  • timer：定时任务服务（替代旧版crontab的部分功能），精准控制任务执行时间，支持秒级调度、依赖触发等高级特性；
  • mount：挂载服务（系统自动挂载磁盘/分区），管理文件系统挂载规则，可配置开机自动挂载、挂载权限等。

单元文件后缀对应类型（如ssh.service、ssh.socket），systemctl 操作时可省略后缀（如systemctl status ssh 等效于 systemctl status ssh.service）。

7.1 核心前提

  1. 操作系统服务操作（启动/停止/设置自启）需要管理员权限，所有指令前需加 sudo；
  2. 所有指令的核心参数是「服务名」（如 clamav-daemon、ssh），需准确输入；
  3. 旧版 /etc/init.d/ 命令仍可兼容，但优先使用 systemctl。

7.2 查看服务状态

7.2.1 查看单个服务详细状态（推荐）

状态解读（终端第一行输出）：

  • active (running)：服务正常运行；
  • inactive (dead)：服务未启动/已停止；
  • failed：服务启动失败（需查看日志排查）；
  • enabled：开机自启已开启；disabled：开机自启已关闭。

9. 文本编辑

vi/vim编辑器

三种模式

  1. 命令模式 - 控制光标移动，进行删除、复制等操作
  2. 插入模式 - 输入文本内容
  3. 底行模式 - 保存、退出、搜索等

基本操作

  • vi filename - 打开或新建文件，光标置于第一行首
  • vi +n filename - 打开文件，光标置于第n行首
  • vi + filename - 打开文件，光标置于最后一行首
  • i - 在当前位置前插入
  • I - 在当前行首插入
  • a - 在当前位置后插入
  • A - 在当前行尾插入
  • o - 在当前行之后插入一行
  • O - 在当前行之前插入一行
  • dd - 删除当前行
  • 2dd - 删除2行
  • yy - 复制当前行
  • nyy - 复制当前行及之后n行
  • p - 在当前光标后粘贴
  • u - 撤销操作

查找与替换

  • /text - 向下查找text
  • ?text - 向上查找text
  • n - 查找下一个匹配
  • N - 查找上一个匹配

退出命令

  • :wq - 保存并退出
  • :q! - 强制退出忽略更改
  • :e! - 放弃修改并重新打开文件
