0x01 dirsearch

核心优势：Python 目录扫描器，内置字典丰富，支持递归/多方法/认证扫描

参数速查

参数 作用
-u 目标 URL
-e 指定扩展名（php,jsp,bak,env…）
-w 自定义字典路径
-t 线程数（建议 ≤15）
--delay 请求间隔（秒），防 CC
--random-agent 随机 User-Agent
--exclude-status 排除状态码（404,400…）
--include-status 仅显示指定状态码
-r 递归扫描
--recursion-depth 递归深度
--cookie 带 Cookie 认证扫描
-H 自定义请求头
--method HTTP 方法（PUT/OPTIONS）
--prefixes 路径前缀（/admin,/api…）
-o 输出文件
--format 输出格式（json/csv/html）

执行说明

🔴 P0 — 每次必跑

1. 万能快速扫描

用途：第一轮快速过一遍，覆盖常见扩展名和敏感文件
注意：线程控制在 15 以下，加 delay 避免触发 CC 防护

2. 敏感文件/备份泄露

用途：专找备份文件、配置泄露、源码泄露，SRC 高危漏洞来源

3. Git/SVN 源码泄露

用途：.git/HEAD、.svn/entries 直接拿源码，P0 级漏洞

🟡 P1 — 重要补充

4. 递归深度扫描

用途：发现子目录后自动深入扫描，适合大型站点

5. 带认证扫描

用途：扫描需要认证才能访问的路径，找未授权访问、越权

6. JS/API 端点发现

用途：JS 中藏 API key、接口路径；API 端点直接找未授权接口

🟢 P2 — 深度挖掘

7. 自定义字典爆破

用途：默认字典不够用时，换大字典深度扫
注意：大字典扫描务必控制线程 10 + delay 0.8s

8. 多方法扫描

用途：PUT 上传点、OPTIONS 信息泄露

9. 状态码精细过滤

用途：403 绕过是 SRC 高频漏洞；301/302 可能暴露内部路径

🔵 P3 — 特殊场景

10. WAF/CC 绕过

核心策略：线程 5-10 + delay 0.5-2s + 代理池 + 随机 UA + 伪造 Header

11. 批量目标扫描

用途：SRC 批量资产扫描，一次性跑完所有子域名

12. CMS/框架专项

0x02 ffuf

适用架构：Vue/React + Spring Boot、前后端分离、REST API
核心优势：支持 POST/JSON fuzz、HTTP/2、灵活过滤器，现代架构首选

参数速查

参数 作用
-u 目标 URL（FUZZ 为占位符）
-w 字典路径
-mc 匹配状态码（200,301,401…）
-fs 按响应大小过滤（0=过滤空响应）
-t 线程数
-p 请求间隔（秒），防 CC
-x 代理地址
-X HTTP 方法（POST/PUT/DELETE）
-H 自定义请求头
-d POST 请求体
-o / -of 输出文件 / 输出格式（json）

执行说明

🔴 P0 — 每次必跑

1. Spring Boot Actuator 探测

用途：Actuator 是 Spring Boot 高危漏洞来源，env/heapdump/mappings 泄露敏感信息
注意：Actuator 端点通常有认证，关注 401/403 响应（存在但未授权）

2. Swagger/API 文档探测

用途：Swagger 泄露全部 API 接口、参数类型、认证方式，SRC 高危漏洞

3. 基础 API 端点 fuzz

用途：发现未授权 API 端点、信息泄露接口
注意：现代 API 返回 JSON，用 -fs 0 过滤空响应

🟡 P1 — 重要补充

4. GET/POST 参数 fuzz

用途：发现隐藏参数、未授权参数、SQL 注入点

5. HTTP 方法测试

用途：PUT/DELETE/PATCH 可能暴露越权操作、未授权访问

6. 认证绕过测试

用途：认证绕过是 SRC 高危漏洞，测试未授权访问、越权

🟢 P2 — 深度挖掘

7. IDOR/越权测试

用途：IDOR 是 SRC 最常见高危漏洞，遍历 ID 测试越权访问

8. 参数爆破

用途：发现隐藏参数、未文档化接口

9. 敏感端点组合 fuzz

用途：发现管理后台、文件操作、调试端点等高危接口

🔵 P3 — 特殊场景

10. WAF/CC 绕过

核心策略：线程 3-5 + delay 0.5-1.5s + 代理池 + 随机 UA

11. 结果聚合分析

用途：聚合分析扫描结果，识别高价值目标

0x03 feroxbuster

适用架构：Vue/React + Spring Boot、前后端分离、REST API
核心优势：Rust 高性能、自动校准过滤（–auto-tune）、智能递归、TUI 界面

参数速查

参数 作用
-u 目标 URL
-w 字典路径
-t 线程数
--delay 请求间隔（300ms/500ms）
--random-agent 随机 User-Agent
--auto-tune 自动调整速率避免 WAF（核心参数）
--depth 递归深度（2-5）
--extensions 附加扩展名（js,json,xml…）
--methods HTTP 方法组合（GET,POST,PUT,DELETE）
--filter-status 过滤状态码（404,400,500…）
--filter-size 按响应大小过滤（0=空响应）
--filter-regex 正则过滤（错误页面）
--headers 自定义请求头
--proxy 代理地址
--json JSON 格式输出
-o 输出文件

执行说明

🔴 P0 — 每次必跑

1. 基础 API 递归扫描

用途：递归发现 API 端点，--auto-tune 自动学习响应模式减少误报

2. 静态资源深度扫描

用途：发现 JS 源码、API 路径、敏感文件泄露

3. Spring Boot 端点扫描

用途：Spring Boot Actuator、Swagger、管理端点是高危漏洞来源

🟡 P1 — 重要补充

4. 带认证扫描

用途：扫描需要认证才能访问的 API，找未授权访问、越权

5. 多方法扫描

用途：PUT/DELETE/PATCH 可能暴露越权操作、未授权访问

6. 扩展名组合扫描

用途：扩展名组合扫描发现隐藏接口、配置文件

🟢 P2 — 深度挖掘

7. 深度递归扫描

用途：深度递归发现多层嵌套 API、隐藏目录
注意：深度递归请求量指数增长，delay 500ms + auto-tune

8. 自定义过滤器

用途：精确过滤减少误报，SPA 站点必须过滤 200 响应

9. 大字典扫描

用途：默认字典不够用时，换大字典深度扫

🔵 P3 — 特殊场景

10. WAF/CC 绕过

核心策略：线程 3-5 + delay 500-2000ms + 代理池 + 随机 UA + auto-tune

11. 结果分析与去重

用途：聚合分析扫描结果，识别高价值目标

0x04 httpx

适用场景：SRC 资产筛选、子域名存活探测、指纹识别、高价值目标排序
核心优势：异步高性能、支持多维度提取（Title/CDN/Tech/Status）、输出灵活可管道串联

参数速查

参数 作用
-sc 显示状态码
-cl 显示响应长度
-ct 显示 Content-Type
-title 显示页面标题
-server 显示 Web 服务器
-td 显示技术栈（Wappalyzer 指纹）
-tech-detect 技术栈检测（同 -td）
-ip 显示解析 IP
-cname 显示 CNAME 记录
-cdn 显示 CDN 信息
-asn 显示 ASN 信息
-location 显示重定向目标
-follow-redirects 跟随重定向
-threads 并发线程数
-rate-limit 每秒最大请求数
-timeout 超时时间（秒）
-retries 重试次数
-path 探测指定路径（可多次）
-H 自定义请求头
-json JSON 格式输出（最完整）
-csv CSV 格式输出
-md Markdown 格式输出
-o 输出文件
-silent 静默模式（仅输出 URL）

执行说明

🔴 P0 — 每次必跑

1. 存活子域名探测 + 基础信息提取

用途：从子域名列表中快速筛选存活 HTTP 服务，提取状态码/长度/标题/服务器/IP 等基础信息
注意：-threads 50 -rate-limit 150 控制并发和速率，避免触发 CC 防护

2. CDN/WAF 识别

用途：识别 CDN/WAF 保护，筛选直连 IP 目标（SRC 直连 IP 漏洞价值更高）
注意：CDN 目标直接打意义不大，优先找真实 IP

3. 技术栈指纹识别

用途：识别 Web 框架/中间件版本，快速定位高危技术栈（Spring Boot Actuator、Tomcat Manager、Shiro 等）
注意：-tech-detect 依赖 Wappalyzer 指纹库，部分站点可能识别不出

🟡 P1 — 重要补充

4. 高价值目标筛选

用途：从海量子域名中快速锁定高价值目标（后台、API 文档、大响应）
注意：SRC 中 API 文档泄露（Swagger/OpenAPI）属于高频高危漏洞

5. 敏感路径/端点探测

用途：批量探测子域名的敏感端点，发现 Actuator 未授权、源码泄露、配置文件泄露
注意：敏感路径探测请求数 = 子域名数 × 路径数，控制 -rate-limit

6. 状态码精细分析

用途：状态码精细分析，401/403 可能存在认证绕过，301/302 暴露内部架构

🟢 P2 — 深度挖掘

7. 响应体内容提取

用途：从响应体中挖掘邮箱、API Key、敏感路径、数据库连接串等
注意：-body 会下载完整响应体，请求量大时控制线程 30 + 速率

8. JSON 输出 + jq 聚合分析

用途：JSON 输出配合 jq 进行多维度聚合分析，发现异常和高价值目标

9. 多源资产批量探测

用途：多源合并、非标准端口全面覆盖

🔵 P3 — 特殊场景

10. WAF/CC 绕过

核心策略：线程 10-20 + rate-limit 20-50 + 代理池 + 随机 UA + 伪造 Header

11. 链式管道组合（subfinder + httpx + nuclei）

用途：全自动化流水线，从子域名发现到漏洞扫描一气呵成
注意：流水线请求量大，确保 httpx 层控制好速率
