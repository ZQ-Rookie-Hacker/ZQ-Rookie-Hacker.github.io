关于information_schema库

MySQL 从 5.0.2 版本开始正式引入了 INFORMATION_SCHEMA 数据库。

关键细节

  1. 首次出现版本：5.0.2（2005年发布）
  2. 目的：
    提供符合 SQL 标准（ISO/IEC 9075）的元数据访问方式，替代部分 SHOW 命令（如 SHOW DATABASES、SHOW TABLES）和直接查询 mysql 系统表的方式。
  3. 兼容性：
    所有 MySQL 5.0 及更高版本（包括 5.1、5.5、5.6、5.7、8.0）均默认包含此库。

主机格式 含义 适用场景
'user'@'localhost' 仅本机连接 本地管理用户（安全）
'user'@'192.168.1.100' 固定 IP 连接 特定机器的服务账号
'user2'@'db.server.com' 固定域名
'user'@'192.168.1.%' （也可使用_） 模糊网段连接 企业内网用户（控制一个网段）
'user'@'192.168.1.0/24' CIDR 子网连接 精准网段控制（专业场景）
'user'@'%' 任意主机连接 测试环境（生产慎用）

用 CREATE USER 创建的用户，默认几乎没有实际操作权限，仅具备最基础的登录能力，可执行少量无需特殊权限的操作
例如：

SHOW ENGINES（查看所有存储引擎列表 ）
SHOW CHARACTER SET（查看字符集信息 ）
这类语句，用于了解 MySQL 服务器的基础配置，无需额外权限就能执行。
