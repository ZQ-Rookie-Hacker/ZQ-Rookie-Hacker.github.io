0x01 dirsearch

优先级说明

🔴 P0 — 每次必跑

1. 万能快速扫描

用途：第一轮快速过一遍，覆盖常见扩展名和敏感文件

2. 敏感文件/备份泄露专项

用途：专找备份文件、配置泄露、源码泄露，SRC 高危漏洞来源

3. Git/SVN 源码泄露

用途：.git/HEAD、.svn/entries 直接拿源码，P0 级漏洞

🟡 P1 — 重要补充

4. 递归深度扫描

用途：发现子目录后自动深入扫描，适合大型站点

5. 带认证扫描（登录后台）

用途：扫描需要认证才能访问的路径，找未授权访问、越权

6. JS 文件/API 端点发现

用途：JS 中藏 API key、接口路径；API 端点直接找未授权接口

7. 后台/管理路径扫描

用途：找管理后台入口、默认页面

🟢 P2 — 深度挖掘

8. 自定义字典爆破

用途：默认字典不够用时，换大字典深度扫

9. 多方法扫描

用途：PUT 上传点、OPTIONS 信息泄露

10. 状态码精细过滤

用途：403 绕过是 SRC 高频漏洞；301/302 可能暴露内部路径

🔵 P3 — 特殊场景

11. WAF 绕过

用途：有 WAF/CC 防护时降低被封概率

12. 批量目标扫描

用途：SRC 批量资产扫描，一次性跑完所有子域名

13. 针对特定 CMS/框架

14. 输出格式与报告
