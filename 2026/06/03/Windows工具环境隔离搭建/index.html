字数: 计算中... 阅读时长: 计算中...

适用于 Python 类 CLI 工具（dirsearch、subfinder、nuclei 等），实现全系统快捷调用、环境隔离、无冲突。

目录结构

核心原则：

  • 外层目录 = 项目根目录，放 bat 脚本 + 虚拟环境
  • 内层目录 = 源码目录，由 git clone 生成
  • bat 脚本强制调用 .venv 中的 Python，彻底隔离全局环境

快速参考

操作 命令
创建虚拟环境 python -m venv .venv
安装依赖 .venv\Scripts\pip.exe install -r <工具名>\requirements.txt
查看调用路径 where <工具名>
更新源码 git pull
重建环境 rmdir /s .venv && python -m venv .venv

注意事项

  1. 不要用全局 pip：始终用 .venv\Scripts\pip.exe
  2. bat 用 %~dp0：不要硬编码路径
  3. PATH 优先级：自定义目录必须在全局 Python 之前
  4. 重开终端：修改 PATH 后必须重新打开 CMD/PowerShell
  5. 虚拟环境不可迁移：换 Python 版本需重建 .venv
