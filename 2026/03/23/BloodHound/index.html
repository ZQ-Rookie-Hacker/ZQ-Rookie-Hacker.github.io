1. 手册概述

BloodHound 是域渗透测试中核心的可视化分析工具，其预构建分析查询可精准定位域内高价值目标、攻击路径、高危权限配置与协议漏洞。本手册系统拆解所有预构建查询的核心用途、实战价值，并给出分阶段使用策略，帮助渗透测试人员高效利用查询结果规划攻击链路。

2. 预构建查询分类详解

2.1 域基础信息类（Domain Information）

查询名称 核心作用 实战价值 备注
Find all Domain Admins 列出Domain Admins组的所有成员 明确域内最高权限账号，锁定渗透最终目标 所有后续查询的核心参照目标
Map Domain Trusts 可视化域信任关系（父子域/外部域/林信任） 规划跨域渗透路径，找到从子域突破到父域的链路 需结合域控拓扑分析信任方向
Find Computers with Unsupported Operating Systems 查找运行已停止支持系统（Win7/2008/XP等）的主机 定位最易突破的初始攻击入口（漏洞多、补丁少） 优先作为初始打点的目标清单

2.2 危险权限类（Dangerous Privileges）

查询名称 核心作用 实战价值 备注
Find Principals with DCSync Rights 查找拥有DCSync权限的主体（用户/组/计算机） 拿到该权限可直接同步所有域账号哈希，控制整个域 域内最高危权限，优先级最高的攻击目标
Users with Foreign Domain Group Membership 查找属于外部域组的用户 作为跨域攻击入口，利用用户在其他域的权限突破边界 需结合域信任关系分析权限传递
Groups with Foreign Domain Group Membership 查找包含外部域成员的组 利用组权限传递特性，实现跨域横向渗透 重点关注高权限组（如管理员组）的外部成员
Find Computers where Domain Users are Local Admin 查找普通域用户为本地管理员的主机 横向移动核心入口，拿到主机权限后窃取高权限凭证 优先对这类主机进行横向打点
Find Computers where Domain Users can read LAPS passwords 查找可读取LAPS密码的主机 直接获取主机本地管理员权限，无需破解 LAPS为本地管理员密码管理工具，是高价值目标
Find All Paths from Domain Users to High Value Targets 生成从普通域用户到高价值目标的完整攻击路径 渗透核心导航，明确“从哪台机、用哪个账号”提权 初始渗透阶段首选查询
Find Workstations where Domain Users can RDP 查找普通用户可RDP的工作站 直接登录后提权、窃取凭证 优先选择核心业务人员的工作站
Find Servers where Domain Users can RDP 查找普通用户可RDP的服务器 服务器承载关键服务，易接触高权限账号 重点关注数据库、应用服务器
Find Dangerous Privileges for Domain Users Groups 查找域用户组的危险特权（如SeImpersonatePrivilege） 利用特权实现本地提权，获取系统权限 结合提权工具（如JuicyPotato）使用
Find Domain Admin Logons to non-Domain Controllers 查找域管理员登录过的非域控主机 窃取这些主机的凭证可直接拿到域管权限 最稳定的提权方式之一，优先分析

2.3 Kerberos 协议类（Kerberos Interaction）

查询名称 核心作用 实战价值 备注
Find Kerberoastable Members of High Value Groups 查找高价值组中可被Kerberoasting攻击的成员 优先攻击高权限服务账号，提升提权效率 高价值组包括域管组、企业管理员组等
List all Kerberoastable Accounts 列出所有带SPN的用户/服务账号 发起Kerberoasting攻击，离线破解NTLM哈希 需结合Impacket等工具请求服务票据
Find Kerberoastable Users with most privileges 查找权限最高的可Kerberoasting用户 精准定位高价值攻击目标，减少无效破解 结合权限图谱分析用户所属组
Find AS-REP Roastable Users (DontReqPreAuth) 查找无需Kerberos预认证的用户 发起AS-REP Roasting攻击，直接获取NT哈希 攻击门槛极低，无前置权限也可执行

2.4 最短路径类（Shortest Paths）

查询名称 核心作用 实战价值 备注
Shortest Paths to Unconstrained Delegation Systems 查找到开启非约束委派主机的最短路径 诱导域管访问后窃取TGT，模拟域管身份 非约束委派为高危配置，优先级极高
Shortest Paths from Kerberoastable Users 查找从可Kerberoasting用户到高价值目标的最短路径 先破解服务账号密码，再横向渗透到域管 需先完成Kerberoasting攻击获取凭证
Shortest Paths to Domain Admins from Kerberoastable Users 定位从可Kerberoasting用户到域管理员的最短路径 直接指向最终提权目标，减少链路分析成本 渗透中期重点执行的查询
Shortest Path from Owned Principals 查找从已控制主体到高价值目标的最短路径 已拿下部分权限后，规划下一步横向移动 “Owned Principals”需手动标记已控制对象
Shortest Paths to Domain Admins from Owned Principals 定位从已控制主体到域管理员的最短路径 直接指向最终提权目标，缩短攻击周期 结合已控主机/用户的凭证信息分析
Shortest Paths to High Value Targets 查找从任意主体到所有高价值目标的最短路径 全局导航，覆盖所有潜在攻击链路 适合整体梳理域内权限拓扑
Shortest Paths from Domain Users to High Value Targets 从普通域用户出发，找到到高价值目标的最短链路 初始渗透核心导航，明确首个攻击方向 与“Find All Paths from Domain Users to High Value Targets”互补使用
Find Shortest Paths to Domain Admins 查找从任意主体到域管理员的最短路径 最直接的提权查询，聚焦最终目标 渗透全阶段均可执行，验证提权路径

2.5 自定义查询（Custom Queries）

  • 用途：编写针对性Cypher查询，满足特殊渗透需求（如精准查找某台主机的权限关系、某用户的登录轨迹等）。
  • 示例场景：
    1. 查找某特定用户的所有权限：MATCH (u:User {name:"DOMAIN\\user1"})-[r:AdminTo|MemberOf*1..]->(t) RETURN u, r, t；
    2. 查找域管登录过的所有主机：MATCH (u:User)-[r:LoggedOn]->(c:Computer) WHERE u:Admin RETURN u.name, c.name。

3. 核心使用策略

3.1 初始渗透阶段（无任何域内权限/仅普通域用户权限）

优先执行：

  1. Find Computers with Unsupported Operating Systems：定位最易突破的初始打点主机；
  2. Find All Paths from Domain Users to High Value Targets / Shortest Paths from Domain Users to High Value Targets：获取全局提权导航；
  3. Find AS-REP Roastable Users (DontReqPreAuth)：发起无门槛的AS-REP Roasting攻击，获取首个凭证。

3.2 已控制部分权限阶段（拿下普通主机/普通用户权限）

优先执行：

  1. Shortest Path from Owned Principals：规划从已控对象到高价值目标的链路；
  2. Find Domain Admin Logons to non-Domain Controllers：查找域管登录过的普通主机，窃取域管凭证；
  3. List all Kerberoastable Accounts：发起Kerberoasting攻击，破解服务账号密码。

3.3 高危漏洞挖掘阶段（需快速提权到域管）

优先执行：

  1. Find Principals with DCSync Rights：查找可同步域哈希的主体，直接控制整个域；
  2. Shortest Paths to Unconstrained Delegation Systems：利用非约束委派窃取域管TGT；
  3. Find Computers where Domain Users are Local Admin：横向移动扩大控制范围，寻找域管凭证。

3.4 Kerberos 攻击兜底阶段（常规路径不通时）

优先执行：

  1. Find Kerberoastable Members of High Value Groups：精准攻击高权限服务账号；
  2. Find AS-REP Roastable Users (DontReqPreAuth)：再次验证低门槛攻击路径；
  3. Shortest Paths to Domain Admins from Kerberoastable Users：利用破解的服务账号密码提权到域管。

4. 实战执行注意事项

  1. 查询结果需结合域内实际拓扑分析，避免依赖单一查询结论；
  2. 标记“Owned Principals”时需准确，否则最短路径查询结果会失真；
  3. 敏感查询（如Find Principals with DCSync Rights）建议在渗透后期执行，避免提前触发告警；
  4. 自定义查询需熟悉Cypher语法，可基于预构建查询修改，满足个性化需求。
